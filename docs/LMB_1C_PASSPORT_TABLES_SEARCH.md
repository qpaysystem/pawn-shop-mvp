# Поиск паспортных данных в таблицах БД 1С

Документ описывает процесс **изучения всех таблиц** БД 1С для поиска колонок, в которых хранятся паспортные данные (серия, номер), и **сопоставление с полями карточки контрагента** (скриншот формы 1С).

---

## 1. Поля карточки контрагента (скриншот) → SQL

| Поле на форме 1С | Пример значения | Таблица.колонка в БД (подтверждённые) |
|------------------|-----------------|----------------------------------------|
| Наименование / ФИО | Фромич Анна Владимировна | `public._reference122x1._fld3178`, `_description` |
| Телефон 1 | 79612209950 | `public._reference122x1._fld41084` |
| Дата рождения | 24.01.1989 | `public._reference122x1._fld3191` |
| Вид документа | Паспорт гражданина РФ | тип значения в форме |
| **Серия** | 50 19 | в этой базе у физлиц часто **не** в _fld3202/_fld3184 (пусто) |
| **Номер** | 961613 | в этой базе у физлиц часто **не** в _fld3201/_fld3185 (пусто) |
| Кем выдан | ГУ МВД РОССИИ ПО НОВОСИБИРСКОЙ ОБЛАСТИ | не установлено |
| Пол | Женский | не установлено |
| Юридический адрес | … | другие _fld* или ссылки |

Для контрагента **Фромич Анна Владимировна** (uid `812a0050569bbedf11eaa55bd5c03527`) колонки `_fld3201`, `_fld3202`, `_fld3184`, `_fld3185` в `_reference122x1` пусты, при этом на форме паспорт заполнен (50 19, 961613). Значит паспорт может храниться в другом месте (хранилище значений, связанная таблица по одной из `*rref`).

---

## 2. Команда поиска по всем таблицам

Используется Artisan-команда **lmb:find-passport-in-1c**: сканирует строковые колонки таблиц БД 1С и ищет вхождение заданной строки (например номер паспорта).

```bash
# Поиск номера паспорта 961613 по таблицам справочников (первые 200)
php artisan lmb:find-passport-in-1c --prefix=_reference --search=961613 --limit-tables=200 --sample

# Только таблицы контрагентов (_reference122*)
php artisan lmb:find-passport-in-1c --prefix=_reference122 --search=961613 --sample

# Другой поиск (серия)
php artisan lmb:find-passport-in-1c --search="50 19" --prefix=_reference --limit-tables=100

# Список таблиц и колонок без поиска
php artisan lmb:find-passport-in-1c --dry-run --prefix=_reference122
```

**Параметры:**

| Параметр | Описание |
|----------|----------|
| `--search=961613` | Строка для поиска (номер или серия паспорта). |
| `--prefix=_reference` | Ограничить таблицы префиксом (_reference, _document, _accumreg и т.д.). |
| `--limit-tables=N` | Максимум таблиц (0 = все). |
| `--sample` | Вывести пример строки, где найдено. |
| `--dry-run` | Только вывести список таблиц и колонок, поиск не выполнять. |

Команда для каждой таблицы выполняет **один** запрос (OR по всем строковым колонкам); при наличии совпадений дополнительно определяет, в какой колонке найдена строка.

### Полный скан по нескольким подстрокам за один проход

**`php artisan lmb:scan-1c-fulltext`** — обходит **все** таблицы `public` (~7852), в каждой строке собирает условие `ILIKE` по всем подстрокам и всем строковым колонкам **одним** запросом; если в таблице есть совпадения, пишет в отчёт пары `колонка → подстрока → число строк`.

```bash
# Дефолтный набор подстрок (ФИО, УФМС, адрес, серия/номер, город)
php artisan lmb:scan-1c-fulltext --output=storage/logs/scan.log

# Свои подстроки (несколько --term=)
php artisan lmb:scan-1c-fulltext --term=999652 --term="Отделом УФМС" --output=storage/logs/my_scan.log

# Без широких 5012 и 999652 (меньше ложных совпадений в телефонах и т.п.)
php artisan lmb:scan-1c-fulltext --skip-broad

# Ограничение для отладки
php artisan lmb:scan-1c-fulltext --limit-tables=500
```

Долгий прогон удобно запускать в фоне: `nohup php artisan lmb:scan-1c-fulltext --output=storage/logs/lmb_1c_fulltext_scan_FULL.log >> storage/logs/lmb_1c_fulltext_scan_console.log 2>&1 &`

## 3. Типы таблиц 1С (на что смотреть)

| Префикс | Назначение | Паспорт может быть |
|---------|------------|---------------------|
| `_referenceNNN`, `_referenceNNNx1` | Справочники | В расширении x1: _fld3201, _fld3202 или др. |
| `_documentNNN` | Документы | Редко (паспорт обычно у контрагента). |
| `_accumreg*` | Регистры накопления | Маловероятно. |
| Регистры сведений, хранилища значений | Разные имена в разных версиях 1С | Возможно, если паспорт вынесен в составной тип. |

В конфигурации «Ломбард» реквизит контрагента **ДокументУдостоверяющийЛичность** в метаданных описан как **Строка(250)** — т.е. один столбец; в БД он может соответствовать другой колонке, не _fld3201/_fld3202.

---

## 4. Результаты сканирования (пример)

После запуска:

```bash
php artisan lmb:find-passport-in-1c --prefix=_reference122 --search=961613 --sample
```

для набора таблиц `_reference122*` строка «961613» **не** найдена ни в одной строковой колонке — что согласуется с тем, что у Фромич в _reference122x1 паспорт в _fld3201/_fld3202 не хранится.

Полное сканирование всех таблиц с префиксом `_reference` (сотни таблиц) выполняется несколько минут; для поиска по всей БД можно по очереди запускать команду с разными `--prefix` (_reference, _document, _accumreg и т.д.) и при необходимости увеличивать `--limit-tables`.

---

## 5. Дальнейшие шаги

1. **Запустить поиск по другим префиксам** (например `_document`, регистры), если по _reference ничего не найдено.
2. **Проверить в Конфигураторе 1С**, куда пишется реквизит «Документ удостоверяющий личность» (какой столбец или связанная таблица).
3. Если паспорт хранится в **хранилище значений** или по ссылке из колонки `_fld*rref`, потребуется запрос к соответствующей таблице по этой ссылке (определить по метаданным или схеме БД).

См. также: `docs/LMB_1C_METADATA_LOMBARD.md` (раздел 6 — сопоставление карточки и SQL), `docs/LMB_1C_TABLES_STRUCTURE_AND_SYNC.md` (структура таблиц и синхронизация).

## 6. Таблицы и колонки, похожие на серию/номер паспорта и «кем выдан»

### 6.1. Серия и номер паспорта

| Таблица | Колонки | Назначение |
|---------|---------|------------|
| **`public._reference122x1`** | `_fld3202`, `_fld3201` | Серия и номер паспорта контрагента (в конфиге: `passport_series_column`, `passport_number_column`). В части баз заполнены у ~23,5 тыс. записей; у других (например Фромич) — пусты при заполненной форме. |
| **`public._reference122x1`** | `_fld3184`, `_fld3185` | Альтернативные колонки серия/номер в других редакциях конфигурации. |
| Любая `_referenceNNNx1` | колонки `_fld*` строкового типа | В других справочниках паспортные поля могут иметь иные номера (`_fld*`); соответствие — по метаданным или выгрузке схемы. |

**Итог:** таблицы, «похожие» на хранение серии/номера паспорта — это в первую очередь **расширения справочников** (`_reference*x1`), где известны колонки **`_fld3201`** (номер) и **`_fld3202`** (серия); для контрагентов это **`_reference122x1`**. В этой базе для ряда физлиц эти поля пусты, хотя на форме паспорт указан — значит данные могут дублироваться в другом столбце (например единый Строка(250) для «Документ удостоверяющий личность») или в хранилище значений/связанной таблице.

### 6.2. «Кем выдан» (УВД, МВД, текст выдачи)

- На карточке контрагента поле **«Кем выдан»** (пример: «ГУ МВД РОССИИ ПО НОВОСИБИРСКОЙ ОБЛАСТИ») в метаданных не вынесено отдельным реквизитом; тип «Документ удостоверяющий личность» в выгрузке задан как **Строка(250)**.
- Возможные варианты хранения:
  1. **Один столбец Строка(250)** в расширении справочника контрагентов — туда может попадать и серия/номер, и «кем выдан» (например склеено или в структурированном виде). Номер такого столбца нужно определять по схеме БД или поиску по содержимому.
  2. **Отдельная колонка** в той же таблице `_reference122x1` (или другой `_reference*x1`) — имя будет вида `_fldNNNN`; найти можно поиском по строке «выдан», «УВД» или «МВД».

**Как искать таблицы, где встречается «выдан УВД» / «МВД»:**

```bash
# Поиск подстроки "выдан" по строковым колонкам таблиц _reference
php artisan lmb:find-passport-in-1c --prefix=_reference --search=выдан --limit-tables=500 --sample

# Поиск "МВД" (часто в тексте "кем выдан")
php artisan lmb:find-passport-in-1c --prefix=_reference --search=МВД --limit-tables=500 --sample
```

Если совпадения появятся — в выводе команды будут указаны таблица и колонка; их и можно считать таблицами/колонками, «похожими» на хранение «кем выдан». Если по всем `_reference` не найдено, имеет смысл запустить поиск по префиксам `_document`, регистрам сведений и т.д.

## 7. Выгрузка всех паспортных данных (команда lmb:export-passport-from-1c)

После того как найдены таблицы с паспортом и «кем выдан», можно выгрузить **все паспортные данные** из них одной командой.

```bash
# Данные по одному клиенту (UID — 32 символа hex из 1С)
php artisan lmb:export-passport-from-1c --uid=812a0050569bbedf11eaa55bd5c03527

# Вывести в консоль до 100 записей из каждого источника
php artisan lmb:export-passport-from-1c --limit=100

# Полная выгрузка в CSV (без лимита)
php artisan lmb:export-passport-from-1c --csv=storage/app/passport_1c_export.csv

# Только из расширения контрагентов (серия/номер/кем выдан)
php artisan lmb:export-passport-from-1c --source=ref122 --limit=500

# Только из хранилища значений типа «Документ удостоверяющий личность»
php artisan lmb:export-passport-from-1c --source=vt3220 --limit=500
```

**Источники:**

| Источник | Таблица | Что выгружается |
|----------|---------|------------------|
| `ref122` | `_reference122x1` | uid, ФИО, телефон, серия/номер паспорта (_fld3201/_fld3202 и _fld3184/_fld3185), кем выдан (_fld3197), дата рождения |
| `vt3220` | `_reference122_vt3220x1` | контрагент (uid), ФИО, все строковые колонки хранилища (контактная информация и тип «Документ удостоверяющий личность»; в т.ч. поля с МВД/выдан) |

В vt3220 хранятся и телефоны/адреса, и данные паспорта (где заполнено). В CSV можно отфильтровать строки по подстроке «МВД», «выдан» или по серии/номеру паспорта.
