**Хост:** 192.168.7.250:5432  
**База:** lmb  
**Пользователь:** lmb  

---

## 1. Базы на сервере

| База       | Назначение   |
|-----------|--------------|
| lmb       | Рабочая БД 1С |
| postgres  | Служебная    |

---

## 2. Схемы

- `information_schema` — системный каталог
- `pg_catalog` — системный каталог PostgreSQL
- `pg_toast` — служебная
- **`public`** — **здесь находятся все объекты 1С**

---

## 3. Важно: почему «таблицы не найдены»

Команда `php artisan lmb:db-schema` использует **information_schema.tables**.  
В PostgreSQL эта представление показывает только те таблицы, на которые у текущей роли есть права. У пользователя `lmb` на таблицы 1С может не быть прав в `information_schema`, поэтому список получался пустым.

Фактически в схеме **public** по каталогу **pg_class** находится:

- **~26 112 объектов** (таблицы + индексы и т.п.)
- Таблицы 1С с именами вида: `_acc38`, `_accrg1208`, `_accopt`, `_accrgat01222`, … (внутренние имена платформы 1С)
- Служебные таблицы платформы: `params`, `schemastorage`, `files`, `v8users`, `v8cmsdpwds`, `ibversion` и др.

## 6. Рекомендация

Для просмотра структуры через `lmb:db-schema` команду нужно доработать: при драйвере **pgsql** получать список таблиц из **pg_catalog.pg_class** + **pg_namespace** (как в этом отчёте), а не из **information_schema.tables**. Тогда будут видны все таблицы 1С в `public`.
