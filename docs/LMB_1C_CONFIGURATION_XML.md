# Выгрузка конфигурации 1С (Configuration.xml)

Файл **Configuration.xml** — это выгрузка метаданных конфигурации 1С в один XML-файл (Конфигуратор → Конфигурация → Выгрузить конфигурацию в файл). По нему можно получить **логические имена** объектов и их реквизиты, а не физические имена таблиц в PostgreSQL.

## Как получить такой файл в 1С

1. Откройте **1С в режиме Конфигуратор**.
2. Подключитесь к базе (например LMB).
3. Меню **Конфигурация** → **Выгрузить конфигурацию в файл**.
4. Укажите путь, например: `C:\Users\...\Documents\Configuration.xml`.
5. Дождитесь окончания выгрузки.

В результате получится один большой XML (сотни тысяч строк) с перечислением всех объектов метаданных: справочники, документы, регистры и т.д.

## Найденные в Configuration.xml объекты Ломбарда

| Логическое имя в 1С | Синоним | Назначение |
|---------------------|--------|------------|
| **ЛМБ_СкупкаЦенностей** | Скупка ценностей | Документ скупки. В БД: таблица `_document389x1` (уже используется в интеграции). |
| **ЛМБ_ОперацияПоЗалогу** | Операция по залогу | Документ залоговой операции. В БД: своя таблица `_document*` (уточнить по количеству записей/структуре). |
| **ЛМБ_ВещиВЗалоге** | Вещи в залоге | Регистр накопления (остатки). В БД: одна из таблиц `_accumrg*` (см. `lmb:1c-balances-discovery`). |

Также встречаются: ЛМБ_ЗалоговыеБилеты, ЛМБ_ЗалоговыеВещи, ЛМБ_ПродажаЗалоговогоИмущества, ЛМБ_ИнвентаризацияЗалоговыхВещей и др.

## Сопоставление с таблицами в PostgreSQL

- В **Configuration.xml** — только **логические имена** (ЛМБ_СкупкаЦенностей, ЛМБ_ВещиВЗалоге) и реквизиты по типам.
- В **БД 1С (PostgreSQL)** платформа создаёт таблицы с **внутренними именами**: `_document389x1`, `_accumrg26227` и т.п. Номер (389, 26227) — внутренний ID объекта в конфигурации; в выгрузке в один XML он может быть в виде GUID или в других файлах при выгрузке «в файлы» (в папку).
- Чтобы понять, какая физическая таблица какому объекту соответствует:
  1. Использовать команды приложения: `php artisan lmb:db-schema --table=_document389x1`, `php artisan lmb:1c-balances-discovery --count` — по количеству записей и списку колонок сопоставить с ожидаемым документом/регистром.
  2. Либо выгрузить конфигурацию **в файлы** (в папку): тогда в подпапках по типам объектов будут отдельные XML с именами и иногда с внутренними номерами.

## Использование для интеграции

- По **Configuration.xml** удобно искать все объекты с префиксом **ЛМБ_** и смотреть их реквизиты (Description, Синоним, вложенные Свойства).
- Команда **`php artisan lmb:parse-1c-configuration <путь к Configuration.xml>`** выводит таблицу документов, регистров и справочников ЛМБ_* для сверки с PostgreSQL.
- Подробный разбор метаданных и сопоставление с таблицами БД: **`docs/LMB_1C_METADATA_ANALYSIS.md`**.
- Для интеграции с нашей БД нужны **физические имена таблиц** и **имена колонок** в СУБД — их получаем командами `lmb:db-schema`, `lmb:1c-balances-discovery` и сопоставлением с логическими именами из Configuration.xml.
