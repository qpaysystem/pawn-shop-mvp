# Подключение к видеопотоку MyVMS (Line Server)

Сервер видеонаблюдения: **192.168.7.7**  
Клиент: программа MyVMS (Observer). Подключение по VPN к сети 192.168.7.x.

## Рабочие URL видеопотока

- **Порт 9780** — веб-API и клиент (запросы к `/` отдают 400; список камер `/cameras` по спецификации требует Digest, на практике поток берём с 9786).
- **Порт 9786** — HTTP-потоки (HLS, FLV, MP4). **Basic Auth: all / all.**

### HLS (рекомендуется для браузера и встраивания)

- Основной поток (высокое разрешение):  
  `http://all:all@192.168.7.7:9786/cameras/0/streaming/main.m3u8`
- Второй поток (низкое разрешение):  
  `http://all:all@192.168.7.7:9786/cameras/0/streaming/sub.m3u8`

Если камер несколько, замените `0` на индекс камеры (1, 2, …).

### Встраивание в страницу (HTML5)

```html
<video controls width="640" src="http://192.168.7.7:9786/cameras/0/streaming/main.m3u8" crossorigin="use-credentials"></video>
```

С учётом логина/пароля: либо проксировать поток через свой бэкенд, либо использовать URL с креденшалами (браузер может блокировать из-за CORS/mixed content).

### Проверка из терминала

```bash
# Список кадров HLS (первые строки плейлиста)
curl -s -u "all:all" "http://192.168.7.7:9786/cameras/0/streaming/main.m3u8" | head -15
```

## Скрипты

- `scripts/camera_stream_check.sh` — проверка доступности и вывод рабочих URL потоков (в т.ч. MyVMS на 9786).
